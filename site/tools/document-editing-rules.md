# 文档编辑规范 (Markdown 撰写指南)

本规范旨在规定本站（个人技术博客）所有 Markdown 文档的格式标准，以确保文章在全平台渲染（特别是 VitePress 与 Obsidian）时的视觉质量、代码正确性及排版严谨性。

---

## 1. 标题格式规范

为了保证页面的学术严谨性与排版美观度，所有章节标题必须遵循以下规则：
1. **禁止在 Heading 行（`#` 至 `######`）中使用任何 Emoji 或图标（如 `💡`、`⚠️`、`🎬`、`📋` 等）**。
   * *正确*：`## 1. 主成分分析与法向量估计`
   * *错误*：`## 💡 1. 主成分分析与法向量估计`
2. **标题符号与文字之间必须有且仅有一个空格**。
   * *正确*：`### 2.1 解决方案`
   * *错误*：`###   2.1 解决方案` 或 `###2.1 解决方案`
3. **单页面 H1 标题唯一性**：正文部分最多只能有 **一个** 一级标题（`#` ），作为文章的主标题。所有子章节应从二级标题（`##`）开始。

---

## 2. 示意图绘制规范 (Mermaid)

用图表辅助理解是技术博客的核心。当需要插入流程图或关系图时，应优先使用 **Mermaid 语法** 代替 ASCII 字符画或手绘图：

1. **图型选型依据**：
   * 报文交互、时序逻辑、网络通信生命周期：`sequenceDiagram`。
   * 算法步骤流程、数据传递、逻辑调用链：`flowchart TD` (纵向) 或 `flowchart LR` (横向)。
   * 包含/分层结构（如滑动窗口、内存模型）：`flowchart` 结合 `subgraph`。
   * 类关系与面向对象继承：`classDiagram`。
   * 状态机转移过程：`stateDiagram-v2`。
2. **防文字截断机制**：
   * 节点内的文本必须精简。
   * 如果文本较长，必须使用 `<br/>` 进行主动换行（每行控制在 14 个汉字以内），防止在不同分辨率屏幕上节点边框发生截断。
     * *正确*：`A["加权局部协方差<br/>Sigma 计算"]`
   * 详细的文字原理解释应该写在图下方的正文或引用块中，不要强行塞进 Mermaid 节点内。
3. **特殊字符转义**：
   * 当节点文本包含括号 `()`、冒号 `:`、大括号 `{}` 等特殊字符时，**必须**使用双引号包裹文本，否则会导致 Mermaid 渲染解析失败：
     * *正确*：`A["调用 connect()"]`
     * *错误*：`A[调用 connect()]`

## 3. 数学公式与符号规范 (LaTeX)

本站已集成 MathJax 数学公式渲染器。公式撰写需满足：
1. **行内公式**：使用单个美元符号包裹：`$f(x) = \mathbf{w}^T x + b$`。
2. **独立块级公式**：使用双美元符号包裹，单独成行：
   $$\mathbf{\Sigma} = \frac{1}{k} \sum_{i=1}^k (p_i - \bar{p})(p_i - \bar{p})^T$$
3. **矩阵表示**：建议使用标准 LaTeX 矩阵环境，如 `\begin{bmatrix} ... \end{bmatrix}`。

---

## 4. 代码块与格式检查规范

1. **严禁出现未闭合的代码块**：所有以 ` ```cpp `、` ```python ` 等开头的代码块，必须在结尾有对应的 ` ``` ` 闭合。
2. **规范指定语言标识符**：所有的代码块都必须显式指定语言名称，例如 `cpp`、`python`、`bash`、`typescript` 等，以确保高亮系统正确渲染。
3. **代码与正文空行隔离**：代码块前后应当各留出一行空行，保持排版舒展。
